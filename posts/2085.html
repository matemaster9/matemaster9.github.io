进程

进程的概念

进程：

  • 程序的一次执行过程
  • 一个程序及数据在处理机上顺序执行发生的活动
  • 有独立功能的程序在数据集合上的一次活动
  • 进程实体的运行过程
进程的状态及转换
  • 基本状态（3种）
    • 运行态
    • 就绪态
    • 阻塞态
  • 普通状态（2种）
    • 新建态
    • 终止态

进程的控制

进程的创建

📌4种主要事件导致进程创建

1）系统初始化

2）正在执行的程序执行了创建系统调用

3）用户请求创建新进程

4）一个批处理作业的初始化

允许-个进程创建另-一个进程。此时创建者称为父进程，被创建的进程称为子进程。子进程可以继承父进程所拥有的资源。当子进程被撤销时，应将其从父进程那里获得的资源归还给父进程。此外，在撤销父进程时，必须同时撤销其所有的子进程。

📌进程创建过程的原语

1)为新进程分配一个唯一的进程标识号，并申请一个空白的PCB (PCB是有限的)。若PCB申请失败，则创建失败。

2)为进程分配资源，为新进程的程序和数据及用户栈分配必要的内存空间(在PCB中体现)。注意，若资源不足(如内存空间)，则并不是创建失败，而是处于阻塞态，等待内存资源。

3)初始化PCB，主要包括初始化标志信息、初始化处理机状态信息和初始化处理机控制信息，以及设置进程的优先级等。

4)若进程就绪队列能够接纳新进程，则将新进程插入就绪队列，等待被调度运行。

进程的终止

📌进程终止的条件

1）正常执行完毕退出

2）出现错误退出

3）出现严重错误退出

4）被其他进程杀死

📌进程终止过程的原语

1）根据进程ID，检索出PCB，读取当前进程状态

2）若当前进程正在执行，则停止执行，将处理机资源调度给其他进程。

3）若当前进程存在子孙进程，必须将子孙进程终止

4）将当前进程的资源归还给父进程，或操作系统

5）将PCB从队列中删除

进程的阻塞

📌阻塞进程原语
1)找到将要被阻塞进程的标识号对应的PCB.
2)若该进程为运行态，则保护其现场，将其状态转为阻塞态，停止运行。
3)把该PCB插入相应事件的等待队列，将处理机资源调度给其他就绪进程。

进程的唤醒

📌唤醒进程原语

1)在该事件的等待队列中找到相应进程的PCB。
2)将其从等待队列中移出，并置其状态为就绪态。
3)把该PCB插入就绪队列，等待调度程序调度。

进程切换

📌进程切换过程

1)保存处理机上下文，包括程序计数器和其他寄存器。
2)更新PCB信息。
3)把进程的PCB移入相应的队列，如就绪、在某事件阻塞等队列。
4)选择另一个进程执行，并更新其PCB.
5)更新内存管理的数据结构。
6)恢复处理机上下文。

2.程序段

什么是进程？

answer：

  • 正在运行的一组程序
  • 具有独立功能的程序在数据集合上的一次活动
  • 操作系统进行==资源分配==的基本单位

answer：

  • 轻量级的进程
  • 具有独立的计数器、堆栈和局部变量，可访问内存共享的变量

注：一个程序至少有一个进程、一个进程至少有一个线程

线程的优先级

操作系统采用时分的方式调度运行的线程，优先级决定了线程享用CPU的时间。 Java 中将线程优先级分为： 1~10。 默认优先级为5。thread.setPriority(int)可以动态调整线程的优先级。

优先级越高，拥有CPU时间越长、反之则越短！

线程优先级的设置策略：

针对频繁阻塞的线程，需要设置较高的优先级。针对偏重于CPU的线程，则设置较低的优先级。但是在JVM以及操作系统中可能会忽略对线程优先级的设定。

📖 Java线程优先级的栗子​

package cn.matemaster.concurrency.four;

import java.util.ArrayList;
import java.util.List;
import java.util.concurrent.TimeUnit;

public class Priority {

    private static volatile boolean notStart = true;
    private static volatile boolean notEnd = true;


    static class Job implements Runnable {

        private int priority;
        private long jobCount;

        public Job(int priority) {
            this.priority = priority;
        }

        @Override
        public void run() {
            while (notStart) {
                Thread.yield();
            }

            while (notEnd) {
                Thread.yield();
                jobCount ++;
            }
        }
    }

    public static void main(String[] args) throws InterruptedException {
        List<Job> jobs = new ArrayList<>();

        for (int i = 0; i < 10; i++) {
            int priority = i < 5 ? Thread.MIN_PRIORITY : Thread.MAX_PRIORITY;
            Job job = new Job(priority);
            jobs.add(job);
            // 创建线程、设置优先级
            Thread thread = new Thread(job, "Thread:" + i);
            thread.setPriority(priority);
            thread.start();
        }

        notStart = false;
        TimeUnit.SECONDS.sleep(10);

        notEnd = false;
        for (Job job : jobs) {
            System.out.println("Job Priority : " + job.priority + "\t"+ "Count : " 
                               + job.jobCount);
        }
    }

}

public class DiyThread {

        Thread thread2 = new Thread(new RunnableImpl(), "Runnable-Thread");

中断线程

中断是Java线程中的一个标识位属性，可以通过thread.interrupt()进行中断当前线程，可以利用thread.isInterrupted()判断当前线程是否有过中断操作，可以利用Thread.interrupted()对中断标识位进行复位操作。

package cn.matemaster.concurrency.four;

import java.util.concurrent.TimeUnit;

public class Interrupt {

    public static void main(String[] args) throws InterruptedException {
        Thread sleepThread = new Thread(new SleepRunner(), "SleepThread");
        sleepThread.setDaemon(true);
        Thread busyThread = new Thread(new BusyRunner(), "BusyThread");
        busyThread.setDaemon(true);

        sleepThread.start();
        busyThread.start();

        TimeUnit.SECONDS.sleep(5);

        sleepThread.interrupt();
        busyThread.interrupt();

        System.out.println("SleepThread interrupted is " + sleepThread.isInterrupted());
        System.out.println(" BusyThread interrupted is " + busyThread.isInterrupted());

        SleepUtils.second(2);
    }

    static class SleepRunner implements Runnable {
        @Override
        public void run() {
            while (true) {
                SleepUtils.second(10);
            }
        }
    }

    static class BusyRunner implements Runnable {
        @Override
        public void run() {
            while (true) { }
        }
    }
}

​ 图.中断异常导致的中断标识位复位

==解析==
这里两个线程标识位结果不一致，SleepThread 为 false 而 BusyThread 为true。
Java api中看出，对于大部分抛出 InterruptedException 的方法，在抛出该异常之前，JVM会自动将该线程的标识位复位，所以 sleepThread.isInterrupted() 的结果仍然是 false

线程的暂停、恢复和终止对应着过期的Java API suspend() resume() stop()

import java.text.DateFormat;

public class Deprecated {

suspend和stop方法的缺点：

suspend在暂停线程时，不会释放暂停线程所包含的资源或者锁，这样一来可能导致死锁
stop在终止线程时，不会给终止线程释放资源的时间，这样会导致不确定的状态

这种暂停、恢复、终止的方案，将会被等待-通知替代

安全终止线程

线程的中断标识位可以作为线程终止的一个条件，或者利用布尔变量来控制线程的终止

安全地终止线程有两种方法：

  • 定义 volatile 标志位，在 run 方法中使用标志位控制线程终止
  • 使用 interrupt 方法和 Thread.interrupted 方法配合使用来控制线程终止

​ 图.利用中断标识位|布尔变量控制线程的终止

📖利用中断标识位|布尔变量控制线程的终止

package cn.matemaster.concurrency.four;

import java.util.concurrent.TimeUnit;

public class Shutdown {

    public static void main(String[] args) throws InterruptedException {
        Runner one = new Runner();
        Thread thread = new Thread(one, "OneRunner");
        thread.start();
        TimeUnit.SECONDS.sleep(1);
        // 中断
        thread.interrupt();
        
        Runner two = new Runner();
        thread = new Thread(two, "TwoRunner");
        thread.start();
        TimeUnit.SECONDS.sleep(1);
        // volatile标志位
        two.cancel();
    }

    static class Runner implements Runnable {
        private volatile boolean on  = true;
        private long i;
        @Override
        public void run() {
            while (on && !Thread.currentThread().isInterrupted()) {
                i++;
            }
            System.out.println(Thread.currentThread().getName() + " Count : " + i);
        }
        public void cancel() {
            on = false;
        }
    }
}

线程间的通信

volatile和synchronized

volatile用于修饰字段，告知所有线程对该变量都必须从共享内存中读取，并且所有对于变量的修改都需要同步刷新回共享内存，确保所有线程对该变量访问的可见性

synchronized用于修饰方法和同步块，确保所有程序在某一时刻只有唯一一个线程处于方法或同步块中，保证线程对方法或同步块的可见性和排他性。

synchronized对象锁：

  • 普通同步方法，🔒是当前实例对象
  • 静态同步方法，🔒是当前类的Class对象
  • 同步块，🔒是synchronized括号内配置的对象

等待-通知

表：等待-通知的相关方法

方法 含义
wait() 线程进入等待状态，并释放🔒
wait(long) 超时等待，时间：long
wait(long, int) 更细粒度的等待

📖 等待-通知的示例​

package cn.matemaster.concurrency.four;

import java.text.SimpleDateFormat;
import java.util.Date;

public class WaitNotify {

    static boolean flag = true;
    static Object lock = new Object();

    static class Wait implements Runnable {
        @Override
        public void run() {
            synchronized (lock) {
                while (flag) {
                    try {
                        System.out.println(Thread.currentThread().getName() 
                                           + "\t flag is true. WAITING\t"
                                + new SimpleDateFormat().format(new Date()));
                        lock.wait();
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
                System.out.println(Thread.currentThread().getName() 
                                   + "\tflag is false. RUNNABLE\t"
                        + new SimpleDateFormat().format(new Date()));
            }
        }
    }

    static class Notify implements Runnable {
        @Override
        public void run() {
            synchronized (lock) {
                System.out.println(Thread.currentThread().getName() + "\t hold lock. Notify \t"
                + new SimpleDateFormat().format(new Date()));
                lock.notifyAll();
                flag = false;
                SleepUtils.second(3);
            }

            synchronized (lock) {
                System.out.println(Thread.currentThread().getName() 
                                   + "\t hold lock again. Sleep\t"
                + new SimpleDateFormat().format(new Date()));
                SleepUtils.second(3);
            }
        }
    }

    public static void main(String[] args) {
        Thread waitThread = new Thread(new Wait(), "WaitThread");
        waitThread.start();
        SleepUtils.second(1);
        Thread notifyThread = new Thread(new Notify(), "NotifyThread");
        notifyThread.start();
    }

}

🔎关于wait、notify、notifyAll的执行细节

1）调用方法：wait、notify、notifyAll时，需要给调用对象加锁

2）wait执行后、线程变成WAITING状态，在等待队列中

3）notify、notifyAll执行后回释放锁，让wait的线程有机会返回

4）wait线程返回的前提时调用对象获得锁

📌等待-伪代码

synchronized(对象) {
    while (等待条件不满足) {
        对象.wait()
    }
    条件满足时处理逻辑
}

📌通知-伪代码

synchronized(对象) {
    改变等待条件
    对象.notifyAll()    
}

经典范式的缺点：等待没有时间限制，那么很可能出现超时的情况，会有线程长期等待资源而不得！

管道输入-输出流

🔎管道输入-输出流主要用于线程间的数据传输，并且传输的媒介为内存

4种具体实现：

  • 面向字节
    • PipedOutputStream
    • PipedInputStream
  • 面向字符
    • PipedReader
    • PipedWriter
package cn.matemaster.concurrency.four;

import java.io.IOException;
import java.io.PipedReader;
import java.io.PipedWriter;

public class Piped {

    public static void main(String[] args) throws IOException {
        PipedWriter out = new PipedWriter();
        PipedReader in = new PipedReader();

        // 将输入流和输出流进行连接，否则回抛出异常
        out.connect(in);
        Thread printThread = new Thread(new Print(in), "PrintThread");
        printThread.start();
        int receive = 0;

        try {
            while ((receive = System.in.read()) != -1) {
                out.write(receive);
            }
        } finally {
            out.close();
        }
    }

    static class Print implements Runnable {
        private PipedReader in;

        public Print(PipedReader in) {
            this.in = in;
        }

        @Override
        public void run() {
            int receive = 0;
            try {
                while ((receive = in.read()) != -1) {
                    System.out.print((char) receive);
                }
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}

thread.join()

如果在线程x中执行thread.join()方法，那么线程x必须在thread线程终止之后，才会从join方法返回

方法 说明
thread.join() 在线程thread结束后，才从join方法返回
thread.join(long) 加入超时限制
thread.join(long, int) 更细粒度的超时限制 
package cn.matemaster.concurrency.four;

import java.util.concurrent.TimeUnit;

public class Join {

    public static void main(String[] args) throws InterruptedException {
        Thread previous = Thread.currentThread();
        for (int i = 1; i <10 ; i++) {
            Thread thread = new Thread(new Domino(previous), "Thread-" + String.valueOf(i));
            thread.start();
            previous = thread;
        }

        TimeUnit.SECONDS.sleep(3);
        System.out.println(Thread.currentThread().getName() + " terminate");
    }

    static class Domino implements Runnable {
        private Thread thread;

        public Domino(Thread thread) {
            this.thread = thread;
        }

        @Override
        public void run() {
            try {
                thread.join();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println(Thread.currentThread().getName() + " terminate");
        }
    }
}

ThreadLocal

ThreadLocal是一个以ThreadLocal对象为键，任意类型为值的存储结构。这个存储结构会被附在线程上，一个线程可以根据一个ThreadLocal对象查询绑定在这个线程上的值

package cn.matemaster.concurrency.four;

import java.util.concurrent.TimeUnit;

public class Profiler {

    private static final ThreadLocal<Long> TIME = new ThreadLocal<Long>() {
        @Override
        protected Long initialValue() {
            return System.currentTimeMillis();
        }
    };

    public static void begin() {
        TIME.set(System.currentTimeMillis());
    }

    public static final long end() {
        return System.currentTimeMillis() - TIME.get();
    }

    public static void main(String[] args) throws InterruptedException {
        Profiler.begin();
        TimeUnit.SECONDS.sleep(1);
        System.out.println("Cost-time: " + Profiler.end() + "mills");
    }
}

等待超时

超时等待，即给等待加一个时间期限。那么在等待时间超过限定范围之后，线程自动恢复。

超时时间段为 T ，那么在时间点：now+T 就会超时

定义如下：

  • remaining = T
  • future = now+T

此时需要wait(remaining) 在wait(remaining)返回后执行：remaining = future - now。如果remaining<0 那么表示此时已经超时，反之则没有超时，继续wait(remaining)

📌超时等待-伪代码

public synchronized Object get(long deadline) throws InterruptedException {
    // 截至超时等待时间点
    long future = System.currentTimeMills + deadline;
    // 超时时限剩余
    long remaining = deadline;
    
    // 超时时限仍有剩余、并且结果不满足预期、则继续等待
    while ( (result == null) && remaining > 0 ) {
        wait(remaining);
        remaining = future - System.currentTimeMills;
    }
    // 结果
    return result;
}
