顺序查找

写在前面：顺序查找，指的是基于链表结构的无序键值序列，在查找的过程中我们按照一个一个地顺序遍历符号表中的所有键，并利用equals()方法来寻找与被查找键相匹配的键

链表的存储结构

public class SqSST<Key ,Value> {

    private Node first;
    private int n;

    private class Node{
            Key key;
            Value val;
            Node next;

        public Node(Key key, Value val, Node next) {
            this.key = key;
            this.val = val;
            this.next = next;
        }
    }

头插法生成链表

public void put(Key key,Value val)
{
    if(val == null) {
        delete(key);    // 延时删除操作delete(key,null)
        return;
    }
    // 如果key存在于符号表，则更新val
    for(Node x=first;x!=null;x=x.next){
        if(key.equals(x.key))
        {
            x.val=val;
            return;
        }
    }
    first=new Node(key,val,first);
    n++;
}

链表的遍历，获取key

public Value get(Key key) {
    if (key == null) throw new IllegalArgumentException("key is null"); 
    for (Node x = first; x != null; x = x.next) {
        if (key.equals(x.key))
            return x.val;
    }
    return null;
}

删除符号表的键值（迭代实现）

public void delete(Key key) {
    if (first == null||key == null)
        throw new IllegalArgumentException("SymbolTable is null or key is null");
    Node p = first;
    Node prev = p;
    while (p!=null) {
        if(p == first&&key.equals(p.key)) {
            first = p.next;
            p = null;
        } else if (key.equals(p.key)) {
            prev.next = p.next;
            p = null;
        } else {
            prev = p;
            p = p.next;
        }
    }
}

删除符号表的键值（递归实现）

public void delete(Key k) {
    if(k == null) {
        throw new IllegalArgumentException("key mustn't be null");
    }

}

private Node delete(Node x, Key k) {
    if(x == null) return null;
    if(k.equals(x.key)) {
        n--;
        return x.next;
    }
    x.next = delete(x.next,k);  // 不断的重置匹配k之前键的next指向
    return x;
}

二分查找

写在前面：这里为了实现二分查找，所以必须保持键的有序性。利用一对平行数组，keys和vals分别存储键值，保持Comparable类型的键有序，以及key-val的一一对应。这里以rank()方法为核心

简易的二分查找

public int binarySearch(Comparable[] arr, Comparable k) {
    int lo = 0;
    int hi = arr.length - 1;
    while(lo <= hi) {
        int mid = lo + (hi - lo)/2;
        int cmp = k.compareTo(arr[mid]);
        if		(cmp < 0) hi = mid - 1;
        else if (cmp > 0) lo = mid + 1;
        else              return mid;
    }    
    return lo;
}

平行数组的存储结构

public class BinarySearchST<Key extends Comparable<Key>,Value> {
    private Key[] keys;
    private Value[] vals;
    private int num = 0;
    private static final int DEFAULT_INIT_CAPACITY = 10;

    public BinarySearchST() {
        this(DEFAULT_INIT_CAPACITY);
    }
    
    public BinarySearchST(int capacity) {
        keys = (Key[]) new Comparable[capacity];
        vals = (Value[]) new Object[capacity];
    }
}

核心操作迭代实现：int rank(Key k)

rank操作实现了二分查找，每一个和比较区间[low,high]的中间元素key[mid]比较。以mid为切分，较小时选择左区间，较大时选择右区间。若是恰好命中，则mid就是被查找键所在的位置，最坏情况下没有命中，mid任然保证左区间较小于key右区间较大于key注：比较区间是键的范围不是整个存储数组的范围

rank()具有两个性质：1.如果key在有序符号表中，那么rank的返回值必定是key在数组中的位置也就是符号表中小于key的键的个数。2.如果key不在有序符号表中，那么rank的返回值也一定是比key小的键的个数 rank的返回值范围为[0,num]

public int rank(Key k) {
    // [0, num-1] : num
    int lo = 0, hi = num-1;
    while(lo <= hi) {
        int mid = lo + (hi - lo)/2;
        int cmp = k.compareTo(keys[mid]);
        if		(cmp < 0) hi = mid -1;
        else if (cmp > 0) lo = mid + 1;
        else 			  return mid;
    }
    
    // mid >= lo -> keys[mid] >= keys[lo]: 始终成立
    return lo;
}

核心操作递归实现：int rank(Key k, int lo, int hi)

public int rank(Key k) {
    return rank(k, 0, num-1);
}

private int rank(Key k, int lo, int hi) {
    if(lo > hi) return lo;
    int mid = lo + (hi- lo)/2;
    int cmp = k.compareTo(keys[mid]);
    if		(cmp < 0) return rank(k, lo, mid-1);
    else if (cmp > 0) return rank(k, mid+1,hi);
    else 			  return mid;
}

核心操作：resize(int capacity)

private void resize(int capacity) {
    if(capacity < num) {
        throw new IllegalArgumentException("resize failed!");
    }
    // 扩容
    Key[] tempK = (Key[]) new Comparable[capacity];
    Value[] tempV = (Value[]) new Object[capacity];
    // 转移元素
    for(int i = 0; i < keys.length; i++) {
        tempK[i] = keys[i];
        tempV[i] = vals[i];
    }
    keys = tempK;
    vals = tempV;
}

基于rank实现put操作

public void put(Key k, Value v) {
    // 判空与是否延时删除
    if(k == null) throw new IllegalArgumentException("key is null");
    if(v == null) delete(k);
    // 获取k的位置/小于k的键的个数
    int i = rank(k);
    if(i < num && k.compareTo(keys[i]) == 0) { 
        vals[i] = v; 
        return;
    }
    // 是否扩容
    if(num >= keys.length) resize(2*keys.length);
    // 迁移元素
    for (int j = num-1; j >= i; j--) {
        keys[j+1] = keys[j];
        vals[j+1] = vals[j];
    }
    keys[i] = k;
    vals[i] = v;
    num++;
}

基于rank实现get操作

public Value get(Key k) {
    if(k == null) {
        throw new IllegalArgumentException("key is null");
    }
    // i : key所在的位置/小于key的键的个数
    int i = rank(k);
    if(i < num && k.compareTo(keys[i]) == 0) return vals[i];
    else 									 return null;
}

基于rank实现delete操作

public void delete(Key k) {
    if(k == null) throw new IllegalArgumentException("key is null");
    if(isEmpty()) return;
    
    int i = rank(k);
    
    // k不在有序符号表中
    if(i == num || k.compareTo(keys[i]) != 0){
        return;
    }
    
    // 迁移元素
    for(int j = i; j <= num-1; j++) {
        keys[j] = keys[j+1];
        vals[j] = vals[j+1];
    }
    keys[num-1] = null;
    vals[num-1] = null;
    num--;
    
    // 占比小于四分之一时,缩容
    if(num > 0 && num <= keys.length/4) 
        resize(keys.length/2);
}

rank的互逆操作：Key select(int k)

为了保持和rank操作的互逆性，k的范围限定在[0,num-1]。k表示key在数组的位置，k+1表示key的排名

public Key select(int k) {
    if(k < 0|| k >= num) return null;
    else                 return keys[k];
}

检查：确保符号表的有序性以及rank和select的互逆性

private boolean check() {
    return isSorted() && rankCheck();
}

// are the items in the array in ascending order?
private boolean isSorted() {
    for (int i = 1; i < size(); i++)
        if (keys[i].compareTo(keys[i-1]) < 0) return false;
    return true;
}

// check that rank(select(i)) = i
private boolean rankCheck() {
    for (int i = 0; i < size(); i++)
        if (i != rank(select(i))) return false;
    for (int i = 0; i < size(); i++)
        if (keys[i].compareTo(select(rank(keys[i]))) != 0) return false;
    return true;
}

因为保证了符号表的有序性，那么根据数组的随机存储的优势，获取符号表中的最大、最小元素、以及删除最大、最小元素都相对容易得多

基于rank实现size(lo, hi)操作

public int size(Key lo, Key hi) {
    if (lo == null) throw new IllegalArgumentException("lo is null"); 
    if (hi == null) throw new IllegalArgumentException("hi is null"); 

    if (lo.compareTo(hi) > 0) return 0;
    if (containsKey(hi)) 	  return rank(hi) - rank(lo) + 1;
    else              		  return rank(hi) - rank(lo);
}

public Iterable<Key> keys(Key lo, Key hi) {
    if (lo == null) throw new IllegalArgumentException("lo is null");
    if (hi == null) throw new IllegalArgumentException("hi is null");

    int low = rank(lo);
    int high = rank(hi);
    ArrayList<Key> list = new ArrayList<>();

    if(low > high) return list;
    for(int i = low; i < high; i++) {
        list.add(keys[i]);
    }
    if(containsKey(hi)) list.add(keys[high]);
    return list;
}

二叉查找树

二叉树的递归结构，可以实现二分查找。这样我们可以利用递归特性完成有序符号表的实现树中的每一个结点都存储k-v，而k的有序性保证了二叉树的结构，以及二叉树与有序数组的映射关系。不同的二叉树结构可以投影出相同的有序键列

二叉查找树在存储键值对时，对有序键列的投影。不同的二叉查找树结构可以获取相同的有序键列

上面图示可以看出，二叉查找树可以存储有序的键值对列。那么我们就可以利用二叉树的递归结构特性，实现二分查找算法。首先实现二叉查找树的查找算法

在二叉查找树中查询一个键，只可能存在两种结果：命中（找到k-v，返回v）、未命中（没找到k-v返回null） 我们可以利用二叉树的递归结构得到如下的算法：

1）如果树是空的，则没有命中返回null。

2）如果查找的键与结点的键相等，则命中 返回结点的值。

3）如果查找的键小于结点的键则递归至左子树查询，反之递归至右子树查询。

查找的递归实现get

public Value get(Key k) {
    return get(root,k);
}

private Value get(Node x, Key k) {
    if(x == null) return null;
    int cmp = k.compareTo(x.key);
    if		(cmp < 0) return get(x.left, k);
    else if (cmp > 0) return get(x.right, k);
    else 		 	  return x.val;
}

查找的非递归实现

public Value get(Key k) {
    Node x = root;
    while (x != null) {
        int cmp = k.compareTo(x.key);
        if		(cmp < 0) x = x.left;
        else if (cmp > 0) x = x.right;
        else 	return x.val;
    }
    return null;
}

现在已经可以从二叉查找树中，顺利的查询需要的k-v，那么如何将有序的键列放入二叉查找树的合适位置？ put方法就可以实现，将键列有序的存储在二叉查找树的结构中！

1）如果k-v存在于二叉查找树，则更新结点的val

2）如果k-v不存在于，则将生成k-v新的结点插入二叉查找树（递归实现）

插入的本质：就是当查找到一个不存在于树中的结点，并结束于一个空链接时，我们需要将链接指向一个包含被查找键的新结点！

public void put(Key k, Value v) {
    root  = put(root, k, v);
}

private Node put(Node x, Key k, Value v) {
    if(x == null) return new Node(k, v , 1);
    int cmp = k.compareTo(x.key);
    if		(cmp < 0) x.left  =  put(x.left, k, v);
    else if (cmp > 0) x.right =  put(x.right, k, v);
    else 			  x.val   =  v;
    x.N = size(x.left) + size(x.right) + 1;
    return x;
}

put插入元素的顺序，决定了二叉查找树的形状。那么元素的插入顺序和元素顺序必须保持良好的（均匀）随机性。否则会出现下面这种情况：（二叉查找树变成了链表）而不是像上面具有良好的二叉树形状！

通过二叉树的有序键列，我们可以获得最大键和最小键。根据二叉树的结构，可以知道最大键就是二叉树的最右叶子结点，最小键就是二叉树的最左叶子结点。那么如何通过递归特性实现寻找到最左右叶子结点？

答：从根结点出发，如果左子树为空那么说明根结点就是最小结点。如果右子树为空那么说明根结点就是最大结点

对于一个存在于二叉树的k-v我们很简单的完成了取整操作。但是对于不存在于二叉树的k-v如何完成取整操作？

答：向下取整1.当根结点的键等于取整的键时，表示此时已经命中，root.key就是我们需要的取整结果。2.当根结点的键小于取整的键时，此时如果左子树是空，则之间返回null表示取整失败。如果左子树不是空，则我们可以确定可能存在取整结果，可以继续递归左子树。3.当根结点的键大于取整键时，此时右子树为空的话，则根结点的键就是取整结果，如果右子树不为空的话，我们需要继续递归至右子树，查找是否存在小于等于取整键。

向上取整的情况只是处理递归的顺序相反，思想保持一致！

向下取整key

public Key floor(Key k) {
    if(k == null) throw new IllegalArgumentException("Key is null");
    return floor(root, k);
}

private Key floor(Node x, Key k) {
    // 子树为空的情况
    if(x == null) return null;
    int cmp = k.compareTo(x.key);
    // 命中，返回根结点键
    if		(cmp == 0) return x.key;
    // 小于取整键，递归至左子树操作
    else if (cmp < 0)  return floor(x.left, k);
    // 大于取整键，递归至右子树
    Key res = floor(x.right, k);
    // 判断是否在右子树中找出取整结果
    if(res == null)    return x.key;
    else               return res;
}

向上取整key

public Key ceiling(Key k) {
    return ceiling(root, k);
}

private Key ceiling(Node x, Key k) {
    if(x == null) return null;
    int cmp = k.compareTo(x.key);
    if		(cmp == 0) return x.key;
    else if (cmp > 0)  return ceiling(x.right, k);
    Key res = ceiling(x.left, k);
    if(res == null) return x.key;
    else            return res;
}

根据二叉树的结构特性，根结点的左子树的结点个数记录了小于根结点键的个数，右子树的节点个数记录了大于根结点键的个数。那么我们可以利用左右子树结点的个数，获取指定位置的键，类似于二叉树映射有序数组，获取指定位置的键。

假设我们现在需要寻找排名为k的键，左子树的结点个数为l。当l=k时，表示根结点就是我们需要找的键。当k<l时，表明需要继续递归至左子树查询，如果k>l那我们就需要再在右子树寻找[k-(l+1)]个结点即可。

select第k个元素

public Key select(int k) {
    return select(root, k);
}

private Key select(Node x, int k) {
    if(x == null) return null;
    int t = size(x.left);
    if		(k < t) return select(x.left, k);
    else if (k > t) return select(x.right, k-t-1);
    else            return x.key;
}

获取k的排名

public int rank(Key k) {
    return rank(root, k);
}

private int rank(Node x, Key k) {
    if(x == null) return 0;
    int cmp = k.compareTo(x.key);
    if		(cmp < 0) return rank(x.left, k);
    else if (cmp > 0) return size(x.left)+1+rank(x.right, k);
    else              return size(x.left);
}

在二叉树中我们如何删除最大最小键？因为最大最小键都是叶子结点，删除之后对二叉树的结构影响最小，所以我们从删除最大最小键入手，探究如何对二叉树进行删除操作！

我们上面的二叉树结构可以看出，最小键的特点：一定是二叉树的最左叶子结点，并且该叶子结点左链接为空，那么在删除该叶子结点时，只需要利用该叶子结点的右链接进行补充即可。也就是说：我们可以不断地对二叉树的左子树进行迭代或递归，知道当前结点的左链接为空时，停止迭代或者递归，并且将当前结点的右链接返回。

删除最小键

public void delMin() {
    root = delMin(root);
}

private Node delMin(Node x) {
    if(x.left == null) return x.right;
    x.left = delMin(x.left);
    x.N = size(x.left) + size(x.right) + 1;
    return x;
}

删除最大键

public void delMax() {
    root = delMax(root);
}

private Node delMax(Node x) {
    if(x.right == null) return x.left;
    x.right = delMax(x.right);
    x.N = size(x.left) + size(x.right) + 1;
    return x;
}

二叉树元素的删除是较为复杂的，在删除元素的同时，需要修改二叉树的形状，并且保持二叉树的有序性。从上面的删除最小键，我们可以看出：对于只有单个子结点的子树，或者是没有子结点的子树，我们可以直接利用它的左右子树进行互相补充，但是对于含有双结点的子树，我们如何删除根结点？接下来介绍一下Hibbard删除法

  • 将指向即将被删除结点x的链接保存为t
  • 将x指向它的直接后继结点min(t.right)
  • 将x的右链接指向去除min(t.right)的子二叉树
  • 将x的左链接指向原来t的左链接

删除二叉树中任意键

public void delete(Key k) {
    root = delete(root, k);
}

private Node delete(Node x, Key k) {
    if(x == null) return null;
    int cmp = k.compareTo(x.key);
    if		(cmp < 0) x.left = delete(x.left, k);
    else if (cmp > 0) x.right = delete(x.right, k);
    // Hibbard删除法
    else {
        if(x.right == null) return x.left;
        if(x.left == null) return x.right;
        Node t = x;
        x = min(t.right);
        x.right = delMin(t.right);
        x.left = t.left;
    }
    
    x.N = size(x.left) + size(x.right) + 1;
    return x;
}

获取键集

public Iterable<Key> keys() {
    if (isEmpty()) return new LinkedList<>();
    return keys(min(), max());
}

public Iterable<Key> keys(Key lo, Key hi) {
    if (lo == null) throw new IllegalArgumentException("key is null");
    if (hi == null) throw new IllegalArgumentException("key is null");

    Queue<Key> queue = new LinkedList<>();
    keys(root, queue, lo, hi);
    return queue;
}

private void keys(Node x, Queue<Key> queue, Key lo, Key hi) {
    if (x == null) return;
    int cmplo = lo.compareTo(x.key);
    int cmphi = hi.compareTo(x.key);
    // 实现的是二叉树的中序遍历
    if (cmplo < 0) keys(x.left, queue, lo, hi);
    // lo <= x.key <= hi
    if (cmplo <= 0 && cmphi >= 0) queue.add(x.key);
    if (cmphi > 0) keys(x.right, queue, lo, hi);
}

获取键数

public int size(Key lo, Key hi) {
    if (lo == null) throw new IllegalArgumentException("low_key is null");
    if (hi == null) throw new IllegalArgumentException("high_key is null");

    if (lo.compareTo(hi) > 0) return 0;
    if (containsKey(hi)) return rank(hi) - rank(lo) + 1;
    else              	 return rank(hi) - rank(lo);
}

红黑树

红黑树的数据结构，我们以指向结点链接的颜色来代表结点的红黑！

public class RedBlackTree<Key extends Comparable<Key>, Value> {

    private static final boolean RED = true;
    private static final boolean BLACK = false;
    
    private RedBlackNode root;

    private class RedBlackNode {
        private Key key;
        private Value val;
        private RedBlackNode left, right;
        private boolean color;
    }
    
    public RedBlackNode(Key key, Value val,boolean color, int size) {
        this.key = key;
        this.val = val;
        this.left = left;
        this.right = right;
        this.color = color;
        this.size = size;
    }
}

如何向红黑树中插入元素？红黑树视为自平衡的二叉树，就需要实现平衡所需的旋转操作。而且红黑树只需要两种情况的旋转就可以实现自平衡！下面的图解左旋出现错误，B结点应该为黑色，A结点为红色。

左旋

private RedBlackNode rotateLeft(RedBlackNode parent) {
    if (parent == null) throw new IllegalArgumentException("RedBlackNode is null");
    RedBlackNode child = parent.right;
    parent.right = child.left;
    child.left = parent;
    //---旋转后的颜色变化---
    child.color = parent.color;
    parent.color = RED;
    //-------------------
    child.size = parent.size;
    parent.size = size(parent.left) + size(parent.right) + 1;
    return child;
}

右旋

private RedBlackNode rotateRight(RedBlackNode parent) {
    RedBlackNode child = parent.left;
    parent.left = child.right;
    child.right = parent;
    child.color = parent.color;
    parent.color = RED;
    child.size = parent.size;
    parent.size = size(parent.left) + size(parent.right) + 1;
    return child;
}

红黑树的插入

接下来我利用图解的方式，介绍一下，向红黑树中插入结点的几种情况：

1）向2-结点插入红结点

这种情况指出出现两种形式：

1.向2-结点插入右红点，此时得到的3-结点是非正常的，需要对结点A作左旋转以得到正常的3-结点

2.向2-结点插入左红点，此时得到的是正常的3-结点，不需要做任何操作

下面的图解第一种情况出现错误，B结点应该为黑色，A结点为红色。得到的结果与第二种正常情况结果一致

2）向3-结点插入红结点

1.插入红结点的键在3-结点的两个键之间

2.插入红结点的键比3-结点的两个键都小

3.插入红结点的键比3-结点的两个键都大（这里父结点B的颜色可红可黑）

第一种情况将下层红链接左旋，即可得到第二种情况

第二种情况将上层的红链接进行右旋，即可得到第三种情况

第三种情况，我们将子结点的红变黑，父结点黑变红，这样就可以将4-结点向上层分解

图解的右下角,B结点应该是黑色的！

那么我们应对上面所属的四种非正常情况，将其进行旋转、变色，就可以完成红黑树的红链接向上传递。也就是2-3树对于4-结点的分解。

如果右子结点是红色，而左子结点是黑色。则需要左旋

如果左子结点是红色，而左子结点的左子结点也是红色，则需要右旋

如果左右子结点均是红色，则需要将左右子变黑，父变红

变色、判断结点颜色

private void flipColor(RedBlackNode x) {
    x.color = RED;
    x.left.color = BLACK;
    x.right.color = BLACK;
}

private boolean isRed(RedBlackNode x) {
    if (x == null) return false;
    else           return x.color == RED;
}

应对四种情况的插入操作

public void put(Key k, Value v) {
    root  = put(root, k, v);
    root.color = BLACK;
}


private RedBlackNode put(RedBlackNode x,Key k, Value v) {
    if (x == null) {
        return new RedBlackNode(k, v, RED,1);
    }

    int cmp = k.compareTo(x.key);

    if      (cmp < 0) x.left  = put(x.left,  k, v);
    else if (cmp > 0) x.right = put(x.right, k, v);
    else              x.val   = v;

    // 三个如果（包含四种情况）保证了不会存在连续两条红链接的情况
    if (!isRed(x.left)&&isRed(x.right))    x = rotateLeft(x);
    if (isRed(x.left)&&isRed(x.left.left)) x = rotateRight(x);
    if(isRed(x.left)&&isRed(x.right))      flipColor(x);
    
    return x;
}

红黑树的删除

理解红黑树的删除操作，首先必要理解2-3-4树自顶而下的变换过程！先看一下最小键的删除，因为有序二叉树的删除方式可以建立在删除最小键的操作之上！

为了保证黑色平衡性，必须要保证删除的结点不是2-结点，也就是可以将需要删除的键，不断地自顶向下进行变换传递，直到最小键被传递到树底，这样我们就可以直接删除最小键并且不用担心删除造成的失衡问题！传递的核心目的：让最小键所在的叶子结点变成3-结点或者是临时的4-结点。待删除完成之后，在逐步自底向上分解产生的临时4-结点。

private RedBlackNode moveRedLeft(RedBlackNode x) {

    flipColor(x);

    if (isRed(x.right.left)) {
        x.right = rotateRight(x.right);
        x = rotateLeft(x);
        flipColor(x);
    }
    return x;
}

删除树底部的最小键

private RedBlackNode delMin(RedBlackNode x) {
    if (x.left == null) return null;

    if (!isRed(x.left)&&!isRed(x.left.left)) {
        x = moveRedLeft(x);
    }

    x.left = delMin(x.left);

    return balance(x);
}

public void delMin() {
    if (isEmpty()) throw new NoSuchElementException("BST underflow");

    if (!isRed(root.left) && !isRed(root.right))
        root.color = RED;

    root = delMin(root);

    if (!isEmpty()) root.color = BLACK;

}

红黑树任意键的删除方法设计思路：

我们在自顶向下的查找路径种做与删除最小键相同的操作，那么就可以保证我们删除的当前结点一定是非2-结点。当我们需要删除的结点在树底部时，可以直接将键删除。当待删除的键不在底部时，我们将当前结点与它的直接后继结点进行交换，然后类似于二叉查找树的删除操作，将问题变为在根结点不是2-结点的子树中删除最小键问题。
