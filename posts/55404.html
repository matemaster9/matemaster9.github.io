lambda演示

public class Lambda {
    public static void main(String[] args) {
        new Thread(new Runnable() {
            @Override
            public void run() {
                System.out.println(Thread.currentThread().getName() + " is running !");
            }
        }).start();
    }
}

以下代码与上面的main方法体中的代码等效 😮
这便是应用了lambda表达式的匿名内部类的写法

new Thread(() -> System.out.println(Thread.currentThread().getName() + " is running !")).start();

我们通过这个演示案例可以知道lambda表达式的好处：简化匿名内部类的使用，大大简化了代码冗余的问题！

lambda？

知晓了lambda表达式的好处，那么lambda表达式到底是什么？

通过一个简单的代码示例理解，what is lambda expression?

public class LengthComparatorTest {
    public static void main(String[] args) {
        String[] strings = {"i","am","second","your","you"};
        Arrays.sort(strings,new LengthComparator());
        System.out.println(Arrays.toString(strings));
    }
}

class LengthComparator implements Comparator<String> {

    @Override
    public int compare(String first,String second) {
        return first.length() - second.length();
    }
}

这里我们看见一句关键的代码：Arrays.sort(strings,new LengthComparator()) 这个针对字符串类型长度比较器，目的是为了将“比较代码块” compare 传入到 Arrays.sort 方法中。而lambda表达式就是为了这一目的诞生的。

我们看下面的操作

public class LambdaDemo {
    public static void main(String[] args) {
        String[] strings = {"i","am","second","your","you"};
        Arrays.sort(strings,
                    (String first, String second) 
                    -> {return first.length() - second.length();});
        System.out.println(Arrays.toString(strings));
    }
}

(String first, String second) -> {return first.length() - second.length();} : lambda表达式成功的将比较代码块 compare 传入到 Arrays.sort 方法中。

也得出什么是lambda表达式的答案！😉

lambda表达式：是一个可传递的代码块，可以在后面被执行一次或者多次！

lambda-函数式接口

@FunctionalInterface
public interface Runnable {
    public abstract void run();
}

通过上面的代码我们看见了一个特别的注解：@FunctionalInterface 这便是函数接口的注解！

函数式接口：只有一个抽象方法的接口，在使用这些接口时，就可以提供一个lambda表达式

我们介绍一下函数接口的特点：

（1）函数接口只能有“一个方法”

（2）函数接口可以有默认方法 default-method

（3）函数接口覆盖Object祖先类的方法

一个方法的说法表示是：除了default方法和Object方法之外，我们只能定义一个自己的抽象方法！

@FunctionalInterface
public interface MyFuncInterface {
    void diyMethod();

    default void defaultMethod(){
        System.out.println("函数接口的默认方法");
    }

    @Override
    String toString();
}

接下来演示一下这个lambda表达式实现函数式接口的过程

class FuncInerfaceImpl {
    public static void main(String[] args) {
        /**
         *	匿名内部类实现函数式接口
         * new MyFuncInterface() {
         *		@override
         *		public void diyMethod() {
         *			System.out.prinln("Impl diyMethod!");
         *		}
         *}
         *	lambda代替匿名内部类实现类函数式接口（精简写法）
         *((MyFuncInterface) () -> System.out.prinln("Impl diyMethod!"))
         *
         *
         */        
        MyFuncInterface funcImpl = () -> System.out.prinln("Impl diyMethod!");
    }
}

转换

lambda表达式最大的吸引力就是可以与函数式接口进行转换！

对于那些有着特别用处的函数式接口（Comparator），而不是单纯的需要一个参数和返回值的方法。换而言之：就是想要lambda表达式做些稍微复杂一点的处理 那么就可以为它建立一个函数式接口！

比如之前在ArrayList底层原理分析中，谈到了一个 removeIf(Predicate<? super E>)方法这里就有一个接口

Predicate<T>

@FunctionalInterface
public interface Predicate<T> {
    boolean test(T t);
    
    // default-method、static-method
}

这个接口专门用来传递lambda表达式，所以每个list下的类都可以调用这个方法来应用lambda表达式

// list清空所有元素
list.reomveIf(e -> e = null);

注： lambda表达式不能直接赋值给Object类型变量，因为这里的Object不是函数式接口！

lambda语法及应用

(parameters) -> expressions;
(parameters) -> {...}

上述代码是标准的lambda表达式语法，当然还根据有无参数分为：有参、无参！以及是否存在返回值的情况！

下面我们以上面提到的比较器为例子 👇

// LengthComparator 
(String first, String second) -> {
    int result = first.length() - second.length();
    if(result > 0) {
        return 1;
    } else if (result < 0){
        return -1;
    } else {
        return 0;
    }
}

// 可以推测参数类型的赋值
Comparator<String> comp = (first,second) -> {return first.length() - second.length();}

// 返回类型通过上下文推导出的情况
(String first, String second) -> first.length() - second.length();

lambda表达式还有需要注意的地方：

（1）变量名不能与lambda表达式中的局部变量重名！

（2）lambda表达式不允许在某些分支返回值，而在另外一些分支不返回！

// error
Comparator<String> first = (first, second) -> first.length() - second.length();

// Illgeal
(String first, String second) -> {
    if((first.length() - second.length()) < 0)
        return -1;
}

        list.add(new Corporation("Microsoft",46));

语法糖

对于一些已经存在于Java标准库的方法（或者是已有的方法），那么我们可以利用更加简洁的方式编写lambda。

❗语法糖：
（1）方法引用
（2）构造器引用

接下来我们展示一下，利用已有的方法去直接实现忽略大小写的字符串排序

String[] arr = {"Mate","mate","Master","master","abc","CBD"}
Arrays.sort(arr, String::compareToIgnoreCase); // 等价于利用lambda实现一个忽略大小写的比较器

利用构造器引用，将字符串列表转化为另外一种类型列表

ArrayList<String> names = new ArrayList<>();  //这里省去加入元素的过程
Stream<Person> stream = names.stream().map(Person::new);
list<Person> people = stream.collect(Collectors.toList());

另外方法引用还可以应用在 this 和 super 两个关键字

this::equals;  // 等价于: this.equals(x)
class Greeter {
    public void greet() {
        System.out.prinln("Hello World!");
    }
}

class TimeGreeter extends Greeter {
    public void greet() {
        Timer t = new Timer(10, super::greet);
        t.start();
    }
}

// 这里 super::greet  等价于 super.greet()

变量作用域

lambda表达式的组成部分：() -> {}
1）参数
2）代码块
3）自由变量值：非参数，而且不在lambda表达式内部定义的变量

先看个栗子☕

public void repeatMsg(String msg, int delay) {
    ActionListener listener = event -> {
        System.out.println(msg);
        Toolkit.getDefaultToolkit().beep();
    };
    new Timer(delay, listener).start();
}

repeatMsg("Master", 10000);   // 每隔10秒打印一下Master

这里 msg 就是自由变量，它是在 repeatMsg 方法中定义的参数变量。它是一个不可变量（final String）

为什么这里需要强调 自由变量msg是一个不可变量❔

答：因为在lambda表达式中，自由变量为最终变量，也就是一经初始化将不再允许它在与lambda表达式同作用域内改变。无论是lambda表达式内部还是外部！

也就是说下面两个做法都是error ❎

public void countDown(int start, int delay) {
    ActionListener listener = event -> {
        start--;
        System.out.println(start);   // error: 不允许改变自由变量 start
    };
    
    new Timer(delay, listener).start();
}
public void repeat(String msg, int count) {
    for(int i = 0; i < count; i++) {
        ActionListener listener = event -> {
            System.out.println(i +":" + msg);  // error: 不允许改变自由变量 i
        };
        new Timer(1000, listener).start(); //默认一千次重复
    }
}

处理lambda

前面说的全都是怎么在方法中应用lambda！那么怎么自己编写一个可以应用lambda表达式的方法呢❓

我们一般把代码块包装在lambda表达式中，主要目的：延迟执行！
延迟执行的原因：

  • 在单独线程运行代码
  • 多次执行代码
  • 在算法的合适位置执行代码（Array.sort的比较器）
  • 回调
  • 必要时才运行

再看repeatMsg ☕

这次我们希望可以这样调用 repeatMsg

repeatMsg(1000, () -> System.out.println("Master"));

那么如何设计这样的方法，便是主要考虑内容！请看下面的设计方式⬇️

public void repeatMsg(int count, Runnable action) {
    for(int i = 0; i < count; i++) {
        action.run();
    }
}

我们得出设计一个应用lambda表达式的方法，就需要利用已有的函数式接口或者提供diy的函数式接口
