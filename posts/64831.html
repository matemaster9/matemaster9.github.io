持续性

非持续性连接：假设现在访问https://matemaster.cn/posts/52745.html页面，此页面大概含有60个资源对象

1）华为浏览器的http进程从port:80端口发起一个与 mateamster.cn的TCP连接

2）华为浏览器客户端发出一个https://matemaster.cn/posts/52745.html请求报文

3）Nginx接受请求，从后端数据库获取请求资源封装成响应报文，并返回该响应报文

4）华为浏览器发起tcp连接关闭的请求，Nginx回应，并关闭连接。

5）华为浏览器分析52745.html的引用，并根据引用请求资源，重复上面四步，直至资源请求完毕！

也就是说：此次获取页面资源，大概需要60个tcp连接

持续性连接：则是在上面的第四步发生不同，

状态码

2xx：正常

  • 200 OK：浏览器请求被服务器正常返回
  • 204 No Content：请求正常响应，但是没有资源返回
  • 206 Partial Content：请求正常响应，返回需要的一部分资源

3xx：重定向

  • 301 Moved Permaently：永久重定向
  • 302 Found：临时重定向
  • 303 See Other
  • 304 Not Modified：资源找到，但请求不满足条件
  • 307 Temporary Redirect

4xx：客户端错误

  • 400 Bad Request：客户端请求存在语法错误
  • 401 Unauthorized：服务器需要验证客户端身份
  • 403 Forbidden：不允许请求该资源
  • 404 Not Found：服务器不存在请求的资源

5xx：服务器错误

  • 500 Internal Server Error：服务器执行请求错误
  • 503 Service Unavailable：服务器繁忙

cookie

HTTP协议是一种无状态的协议，简而言之就是，服务器不会记录浏览器的访问请求，即使是刚请求的浏览器，服务器依旧视为第一次请求访问！但是Cookie引入，支持了HTTP状态，第一次华为浏览器访问阿里官网，阿里服务器会在响应报文中加入cookie字段，并且在阿里的后端数据库记录该cookie。同时华为浏览器在来自阿里服务器的响应报文中查找到Set-Cookie: sid = 18967626，并在浏览器的cookie管理中添加这个信息，以便于下次**@matemaster_pc**访问阿里官网时，将cookie加入到请求头中。阿里服务器在华为浏览器下一次访问时，进行数据库检索，如果存在cookie记录，那么阿里将会自动记录该浏览器的浏览记录，保持状态！

web服务器

代理服务器的作用就是做web缓存。web缓存可以减轻接入英特网的压力，并且加快了客户端的响应速度！

这里为了保证web缓存有效期，我们的web缓存服务器，会向源服务器发送GET请求，并且在字段中存在If-Modified-Since与Last-Modified做对比，当不一致时说明缓存过期，那么将更新web缓存！

# 上一次修改时间
Last-Modified: Mon, 29 Mar 2021 05:59:01 GMT
# 假设上一次修改时间
If-Modified-Since: Mon, 29 Mar 2021 05:59:01 GMT

mail

邮件系统的组成：user-agent、mail server、SMTP

邮件的发送过程：以matemaster@163.com向2099625007@qq.com为例

1）首先，matemaster本地的用户代理，会将含有发送方和接收方信息，以及邮件主体信息的邮件报文(SMTP)。并且在25号端口建立与邮件服务器的tcp连接，发送给对应的网易邮件服务器，并且将报文存在服务器的报文队列当中

2）网易邮件服务器，将根据接收方信息，利用tcp连接将邮件报文发送给腾讯邮件服务器(SMTP)

3）腾讯邮件服务器在收到邮件报文之后，主动将报文推送给对应接收方的用户代理(SMTP)

SMTP与HTTP之间区别在于：

①SMTP属于push协议，不可以主动从服务器端获取邮件，只能等待服务器推送！HTTP属于pull协议，可以主动向服务器请求获取资源！
②SMTP每个报文的内容必须是7位ASCII码格式

Wireshark抓包看出SMTP报文信息：（就是matemaster@163.com->user-agent与网易邮件服务器的对话）👇

标准的报文格式：

From: matemaster@163.com
To: matemaster@qq.com
Subject: This is a simple title!

POP3三个工作阶段

1）客户校验authorization：此时主要为了校验用户的身份

2）事务处理：取回SMTP报文、对报文进行删除标记和取消删除标记，获取邮件的统计信息

3）更新：在用户指向quit命令之后，邮件服务器会删除那些被删除标记的报文

DNS

DNS服务

基本服务

DNS提供了域名->ip的解析服务。主要由：DNS分布式数据库以及使得主机可以查询数据库的应用层协议构成。

DNS建立在UDP协议之上进行网络传输，互联网端口port:53

举个栗子✌

从应用层解释：本地主机(@localhost)向服务器请求www.matemaster.cn/index.html资源时，DNS的操作过程

1）web browser:华为浏览器向本地电脑的DNS客户端程序发送获取www.matemaster.cn的ip地址请求

2）本地电脑的DNS客户端向远端DNS server发送上述请求报文

3）DNS server从后端分布式数据库或者DNS缓存中获取ip，并封装成响应报文，返回给DNS客户端

4)web browser:华为浏览器根据DNS客户端返回的ip与远端服务器www.matemaster.cn建立tcp连接

额外服务

除了域名->ip的解析服务，DNS支持：主机别名、邮件服务器别名、负载分配。

主机别名：对于含有复杂hostname的主机，可以支持多别名的操作。而其中存在一个规范主机名和多个主机别名而DNS便是提供，将多个主机别名，转为对应的规范主机名。

邮件服务器别名：为了让邮件用户可以自己定义好记的邮件名，我们就需要利用类似主机别名的方式，为邮件服务器提供别名操作，而在DNS记录中存在的MX记录，允许邮件服务器名和web服务器名相同。

负载分配：简单理解：就是一个IP地址集合与同一个规范主机名相联系！（可以参考下面在Ubuntu系统下的nslookup alibaba.com）。从那里看出，一个alibaba.com对应了4个IP地址

DNS分布式数据库

分布式数据库结构

DNS采取的是：分布式、层次数据库。层次：root、TLD、权威

看看例图✌

root server：提供了对应顶级域服务器的ip

TLD：顶级域服务器提供了存储了主机ip的权威服务器的ip

权威服务器：存储了对应主机名的ip地址

本地DNS server：虽然不属于分布式层次结构中的数据库，但是在DNS中起重要作用。与DNS缓存有关！

alibaba.com的DNS解析过程：

1）@matemaster_pc向本地DNS服务器发送请求，以获取alibaba.com域名的ip地址

2）本地DNS服务器先查询数据库，看是否存在对应ip地址的缓存。

  • 若存在直接返回给主机

  • 若不存在则，向根服务器发送请求

3）响应报文，获取com顶级域服务器的ip地址

4）向com顶级域发送请求报文

5）响应报文，获取alibaba.com权威服务器的ip地址

6）向权威服务器发送请求

7）响应报文，获取alibaba.com主机的ip地址

8）返回给@matemaster_pc

DNS记录

DNS记录包含4元组：[name,value,type,TTL]

type的值，决定了name-value之间的关系

type = A：name是主机名，value是主机名对应的IP地址

type = NS：name是域，value也是主机名——该域主机IP对应的权威服务器

type = CNAME：name是主机别名，value是对应的规范主机名

type = MX：name邮件服务器别名，value是对应的规范主机名

（1）当某个DNS服务器server_it不是alibaba.com的权威服务器，那么server_it必将包含两条记录：A和NS

NS(alibaba.com,ns1.alibabadns.com,NS,100)

A(ns1.alibabadns.com,198.11.138.254,A,100)

（2）当某个服务器server_ali是alibaba.com的权威服务器，那么server_ali一定含有A记录

A(alibaba.com,203.119.215.82,A,100)

比如我在华为云域名服务器注册的域名matemaster.cn，在DNS系统数据库中至少存在两个关于我域名的记录：(matemaster.cn,ns1-huaweiclound-dns.net,NS)、
(ns1-huaweiclound-dns.net,198.70.153.255,A)

报文参数

Transaction ID：事务ID，只是单纯的2B，16bit的数据，用于标识报文的唯一性

Flags：

  • QR(Response) 1bit：标记是请求还是响应报文

    • 0：请求
    • 1：响应

  • Opcode 4bit：

    • 0：标准查询
    • 1：反向查询
    • 2：服务器状态请求

  • Authoritative 1bit：标记发送此响应报文的服务器是否为权威服务器

    • 0：
    • 1：是权威服务器

  • Truncated 1bit：数据是否被截断，在数据超过512B时自动截断，也就是只能传输512B

    • 0：
    • 1：

  • Recursion desired 1bit：是递归查询还是迭代查询

    • 0：

  • Recursion available 1bit：只出现在响应报文，表明服务器是否支持递归查询

    • 1：

  • Z 1bit：保留字段只能为0

  • Answer authenticated 1bit：服务器授权验证

    • 0：
    • 1：

  • Non-authenticated data 1bit：保留字段，只能为0

  • Reply code 4bit

    • 0：无差错
    • 1：格式差错
    • 2：问题在域名服务器上
    • 3：域参照问题
    • 4：查询类型不支持
    • 5：在管理上被禁止
    • 6 ~ 15 保留

public class ServerSocketDemo {
    public static void main(String[] args) throws IOException {
        ServerSocket serv= new ServerSocket(8189);
        Socket client = serv.accept();
        InputStream in = client.getInputStream();
        OutputStream out = client.getOutputStream();
        Scanner sc = new Scanner(in, "UTF-8");
        PrintWriter output = new PrintWriter(new OutputStreamWriter(out, "UTF-8"), true);
        output.println("Hello Enter BYE to exit");
        boolean done = false;
        while(!done&&sc.hasNextLine()) {
            String line = sc.nextLine();
            output.println("Server : "+line);
            if(line.trim().equals("BYE")) {
                done = true;
            }
        }
    }
}
