netID hostID srcIP dstIP 作用

子网掩码

固定三级IP的概念 IP:{<net><netChild><host>}

要求做到两件事

  1. 可用根据IP地址和子网掩码求出子网地址
  2. 可用根据IP地址和子网掩码求出可用网络范围
IP 145.13.3.10
子网掩码 255.255.255.0
子网地址 145.13.3.0
最小可用IP 145.13.3.1
最大可用IP 145.13.3.254

子网划分的两种情况：

  1. 给定网络地址。已知需求的子网个数以及主机个数、求划分子网的策略

    eg：假设现在已经有一个IP地址：192.168.10.0，现在需要划分出来4个子网主机有25台，求如何划分子网？

    resolve：

    1. 根据主机数求出主机号需要多少位、以及子网需要借位多少

    子网的可用主机数 host_number=2^n-2 if(host_number>=25) n=5,也就是需要借三位主机号作为子网号。

    1. 根据借位得出子网掩码、子网地址

    得子网掩码：11111111.11111111.11111111.11100000 = 255.255.224.0 相与得子网地192.168.10.(xxx|host)

    1. 得出可用子网号

      得子网号：001 010 100 101 110 (111和000不可用)得到可用起始网络IP

    2. 根据子网规则划分出可用得IP范围

    [192.168.10.32] 192.168.10.33~192.168.10.62

    [192.168.10.64] 192.168.10.65~192.168.10.94

    [192.168.10.128] 192.168.10.129~192.168.10.158

    [192.168.10.160] 192.168.10.161~192.168.10.190

    [192.168.10.192] 192.168。10.193~192.168.10.222

  2. 不给定网络地址。已知需求的子网个数以及主机个数、求划分子网的策略

    同上、在选择子网时主要按照ABC三类得基础之上划分子网

子网下得路由器转发表

Net 子网掩码 下一跳路由器
net1 x.x.x.x route1
net2 x.x.x.x route2
net3 x.x.x.x route3

子网下得分组路由转发算法

  • 1 提取srcIP
  • 2 与路由器直接相邻得网络，srcIP相与其子网掩码查看是否存在该srcIP。
    • 存在时，直接交付给相应得网络
    • 不不存在，则继续执行下一步
  • 3 查看路由器得转发表
    • 若存在相应srcIP，则交付给相应得下一跳路由器
    • 不存在，则继续执行下一步
  • 4 将表中得子网掩码和srcIP相与，查看是否匹配
    • 若匹配，则交付给相应得下一跳路由器
    • 不匹配，则继续执行下一步
  • 5 查看是否路由表是否有默认路由
    • 有，则交付给默认路由器
    • 没有，则直接分组转发报错

路由器的网关地址

在网络中具有路由功能的设备必须配备网关地址：相当于一个网络的出口和入口。而这一个IP地址必须要和路由器连接的主机的IP在同一子网下。其他主机的IP分组通过这一网关地址进入子网中查找相应的主机，该子网内的主机通过这一网关地址发送出IP分组。

IP多播与D类地址

为了实现多播的功能、由于不可能把所有IP地址放入多播数据包中、所以采取将多播数据包的目的IP地址写入D类地址中进行传输。使用特定的D类地址作为目的IP、实现一次播出多个主机接收的目的

路由协议

路由算法

LS-贪心

起点为s、终点v

表达式 含义
D(v) s-v的费用
p(v) s-v最短路径上、v的前续节点
N s-v上已经确定的节点集合
c(x,y) 任意两个节点之间的距离 
init:
	N = {s}
	for all nodes in map
        if v is the neighbor of s
            then D(v) = c(s,v)
        else D(v) = ∞
    
loop:
	find w not in N such that D(w) is a minimun
        add to N
    update D(v) for each node that is the neighbor of s
        D(v) = min{D(v), D(w)+c(s,w)} 
        
until: N contains all nodes

步骤 N D(B),p(B) D(C),p(C) D(D),p(D) D(E),p(E) D(F),p(F)
0 A 1,A ∞ 1,A 2,A 3,A
1 AB 3,B 1,A 2,A 2,B
2 ABD 3,B 2,A 2,B
3 ABDE 3,B 2,B
4 ABDEF 3,B
5 ABDEFC
A-B=1
A-B-C=3
A-D=1
A-E=2
A-B-F=2

算法思想：

把网络中的节点分为两个集合N N`

N：代表已经确定了最短路径的节点

N`:代表还没有确定顶点

初始状态下：只存在起点A一个

随后：每次挑选开销最小的路径节点加入N、并且更新相应的路径

直到N`为空

DV-动态规划

距离向量方程式Bellman-Ford

距离向量x-y：找出x的最小邻点

initial:
	for all dstNode y in N
		if y is the neighbor of x
			then d(xy) = c(x,y)
		else d(xy) = c(x,y) = ∞
    for each neighbor w
    	d(wy) = ? for all dstNode y in N
    	send distance vector Dx=[d(xy): y in N] to w
    	
loop:
	wait until a link cost change to some neighbor w or until I receive a distance verctor from some neighhbor w 
	for each y in N
		d(xy) = min [c(x,v)+ d(vy)]
	if d(xy) changed for any dstNode y in N
	 	then send distance vector Dx=[d(xy): y in N] to all neighbors
	 
forever

算法思想：

对于网络中的任意一个节点x：需要维护下面的信息

  1. x到所有邻点v的距离：c(x,v)
  2. x到图中其他节点的距离向量：Dx=[d(xy): y in N] 也是x到任意其他节点y的距离估计值
  3. 对于x任意邻点v自己的距离向量、Dv=[d(vy): y in N]

在距离向量算法中、每个节点都向自己的邻点发送自己的距离向量、然后邻点根据发来的距离向量进行更新自己的距离向量

分层次自治系统

为什么要对网络划分层次？

  1. 互联网庞大、要想维护所有网络节点的信息、路由表太大无法管理
  2. 很多内部网络私有化、不会对外公开

  1. 基于链路状态——Dijikstra算法
